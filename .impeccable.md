## 设计上下文 (Design Context)

### 用户与目标 (Users)
此环境专供开发者个人使用。主要目标是为长时间的编程工作和 Markdown 写作提供一个专注、不易疲劳的工作空间。

### 品牌性格 (Brand Personality)
终端环境的性格由三个词定义：**简约**、**专注**、**禅意**。设计深受 Obsidian.md 的深邃、低饱和度、现代美学影响。
**反面参考 (Anti-references)**：必须严格避免使用鲜艳、刺目或霓虹色彩。

### 审美方向 (Aesthetic Direction)
当前设计的绝对重点是暗色模式 (`appearance: dark`)，采用极深的炭黑背景 (`#121212`) 和高对比度但柔和的浅灰前景 (`#E0E0E0`)。调色板依赖于去饱和的非霓虹 ANSI 颜色（如薄荷绿、晴空蓝、薰衣草紫）。未来保留设计亮色模式的可能性，但当前不作为重点。

### 设计原则 (Design Principles)
1. **沉浸与专注**：对比度和视觉元素的调整必须以最大化使用者的沉浸感和专注力为首要目标，尽可能减少视觉干扰。
2. **视觉舒适**：优先使用低饱和度色彩，避免刺眼的高亮度，以防止长时间使用导致的视觉疲劳。
3. **禅意极简**：保持界面整洁，摒弃不必要的装饰。每一种颜色和 Token 都必须有清晰的语义，不能对使用者造成信息过载。
4. **语义一致性**：严格遵循已定义的语义 Token (`tokens.text`, `tokens.intent`)，确保在所有终端应用（如 Ghostty、Fish 等）中维持统一的视觉语言。
